CharacterLibraryBuilder

Su trabajo sería:

guardar una biblioteca de piezas 3D
mostrarlas en una interfaz
dejar arrastrarlas al personaje
validar si encajan

1. Qué es este módulo realmente

Piensa en este módulo como una mesa de trabajo digital.

A un lado tienes la biblioteca:

cabezas
cabellos

brazos

ropa
accesorios

En el centro tienes el personaje base.

Y el usuario hace esto:

toma una pieza
la arrastra
la suelta sobre el personaje
el motor la equipa automáticamente

Eso es el sistema.

2. Qué partes debe tener el módulo

Este módulo debe dividirse en 8 partes.

A. Biblioteca de assets

Guarda todas las piezas disponibles.

B. Base de datos de piezas

Guarda información de cada pieza.

C. Vista previa 3D

Muestra el personaje armado.

Panel con miniaturas, pestañas y zonas de drop.

Controla arrastrar y soltar.

Quita y pone piezas.

G. Validador de compatibilidad

Revisa si una pieza sí puede usarse.

H. Guardado y carga

Guarda combinaciones en JSON o base de datos.

9. Cómo funciona el drag-and-drop

Aquí está el corazón del sistema.

Paso 1: seleccionar pieza

El usuario hace clic en una miniatura.

Paso 2: arrastrar

La interfaz crea un “drag object”.

Paso 3: mover

Mientras mueve la pieza, el sistema resalta zonas válidas.

Ejemplo:

cabello → resaltar cabeza
zapatos → resaltar pies
capa → resaltar espalda
Paso 4: soltar

Cuando la suelta encima del personaje:

detecta la zona
detecta la categoría
valida compatibilidad
reemplaza la pieza actual
actualiza la vista
10. Qué son las zonas de drop

El personaje debe tener zonas lógicas, como sockets o puntos de anclaje.

Cuando arrastras una pieza, el sistema pregunta:

“¿Esta pieza pertenece a este socket?”

Si sí, la deja.
Si no, la rechaza.

CharacterLibraryBuilder.

Objetivo:
Crear una biblioteca de piezas 3D modulares y un sistema visual drag-and-drop para ensamblar personajes dentro del motor.

El módulo debe hacer lo siguiente:

1. Mostrar una ventana o panel llamado Character Builder.
2. Cargar un personaje base 3D en una vista previa.
3. Cargar una biblioteca de assets modulares desde carpetas o metadata.
4. Organizar las piezas por categorías:
   - body
   - head
   - hair
   - torso
   - arms
   - legs
   - shoes
   - outfit
   - accessory
5. Mostrar miniaturas de cada asset en la interfaz.
6. Permitir arrastrar una miniatura y soltarla sobre el personaje 3D.
7. Detectar el socket o zona de drop correcta.
8. Validar compatibilidad por:
   - category
   - skeleton_id
   - body_type
   - attachment_socket
9. Si la pieza es compatible, reemplazar automáticamente la pieza actual de esa categoría.
10. Si la pieza no es compatible, mostrar mensaje de error.
11. Mantener todas las piezas unidas al mismo esqueleto base.
12. Permitir rotar y hacer zoom en la vista previa.
13. Permitir guardar el personaje armado como preset en JSON.
14. Permitir cargar presets guardados.
15. Permitir resetear el personaje al estado base.
16. Incluir opción random para equipar piezas aleatorias válidas.
17. Incluir sistema de filtros por categoría, tags y tipo de cuerpo.
18. Diseñar el sistema como módulo desacoplado y reutilizable.

Quiero dividir el sistema en estas partes:
- CharacterLibraryBuilder
- AssetLibrary
- AssetMetadataDatabase
- CharacterAssembler
- DragDropController
- CompatibilityValidator
- PreviewViewport
- PresetManager
- UICharacterBuilder

Quiero que si faltan funciones del motor se creen wrappers abstractos con nombres claros para conectarlos después a la API real.
